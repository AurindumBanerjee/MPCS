Technical Research Report · System Design

Malleable Perceptual-Cognitive System

A dual-process AI framework for adaptive perception, salience-driven reasoning, and experience-based learning
📄 Research Paper 🔬 AI Architecture 📅 2025 🏷 MPCS · Dual-Process · Cognitive AI
Abstract

We present the Malleable Perceptual-Cognitive System (MPCS), a unified AI framework that mirrors how biological minds selectively allocate cognitive resources. MPCS addresses a fundamental gap in conventional AI: the inability to distinguish between situations demanding instant reflexive action and those warranting careful deliberation. The system integrates multisensory, malleable input representations, a salience-and-threshold routing mechanism, dual processing pathways (reflex and deliberative), a two-layer memory architecture, and an experience-driven adaptive learning loop. The central design principle is captured in a single idea: not all stimuli require equal thinking. We describe the system's architecture, constituent components, processing pipeline, and avenues for future extension.

§ 01

Introduction

When a person hears a sudden loud bang, they flinch before consciously registering the sound. When the same person encounters an unfamiliar object in a dark room, they slow down, look carefully, and reason through the situation. These two behaviors reveal a profound architectural truth about intelligent systems: cognition is not uniform. The brain operates across a spectrum, from near-instantaneous reflex arcs to slow, deliberate reasoning chains.

Conventional AI systems process all inputs through a single, uniform pipeline. A neural network applies the same forward-pass computation to a trivial query and a safety-critical decision alike. There is no notion of urgency, no reflexive short-circuit, no emotional context shaping how much attention the system pays. This uniformity is not merely inefficient — it is architecturally misaligned with how intelligent biological systems actually work.

The core problem

Traditional AI architectures lack three capabilities that define biological intelligence:

Gap 1
No reflexive pathways
Every input goes through full computation regardless of urgency. There is no fast lane for high-stakes, time-sensitive stimuli.
Gap 2
No emotional context
Systems ignore valence, arousal, and emotional significance — the very signals that determine priority in biological perception.
Gap 3
No adaptive memory
Responses do not improve from experience within the same perceptual context. Memory is static, not experientially updated.

Objective

MPCS is designed to fill all three gaps. It provides a principled architecture that perceives the world in a rich, multimodal way, evaluates how urgent each stimulus is, routes it appropriately — either to a fast reflex response or a slow deliberative reasoning process — and then learns from the outcome to improve future responses.

System objective: Perceive → Evaluate Salience → Route (Reflex | Deliberation) → Act → Learn

This pipeline is not a metaphor — each stage corresponds to a concrete computational module described in detail throughout this report.

§ 02

Background Concepts

MPCS draws on four conceptual building blocks from computer science and cognitive science. Understanding each of them provides the foundation for understanding the system as a whole.

2.1 Bloom Filters — efficient memory

A Bloom filter is a probabilistic data structure that answers the question: have I seen this before? It does so in constant time and with minimal memory by hashing an item across multiple bit arrays. It has one important asymmetry: it can have false positives (claiming to have seen something it has not), but it produces no false negatives (it never claims something is unseen when it was actually seen).

Intuition

Think of a Bloom filter as a very fast, very compact bouncer at the door of memory: it can definitively say "no, I've never seen that" but its "yes, I've seen that" might occasionally be wrong. For perception systems, a false positive (treating something familiar as novel) is much less harmful than a false negative (treating something novel as familiar).

In MPCS, Bloom-filter-style lookup is used in the fast membership layer of the memory system to rapidly determine whether an incoming stimulus has been encountered before — a prerequisite for computing the novelty component of salience.

2.2 Neural data structures — novelty detection

Biological nervous systems encode sensory experience not as raw data, but as compressed signatures — highly reduced representations that capture the most discriminative features of a stimulus. These signatures enable rapid comparison: is this new stimulus similar to anything in memory? Neural data structures in the MPCS context follow the same principle: stimuli are embedded into a compact feature space, and novelty is measured as distance from the nearest stored signature.

2.3 Multisensory data structures

Real-world perception is inherently multimodal. A threat is perceived not just visually, but through sound, body tension, environmental context, and past experience. A multisensory data structure bundles all these modalities into a single structured unit — a perception record — together with derived features (intensity, emotional tag, confidence) and metadata (time, location, context). This unification is what enables the system to reason holistically about a stimulus rather than treating each sensory channel in isolation.

2.4 Z* numbers — emotion and uncertainty

Z* numbers extend classical fuzzy sets to represent both uncertainty and emotional confidence simultaneously. In MPCS, they provide the mathematical idiom for encoding a percept:

Z*(S) = ⟨ Context, Time, Subject, Belief, Emotion ⟩ → Belief: how confident the system is about what it perceives → Emotion: the affective valence and arousal attached to the percept

This formalism is what allows the MPCS salience evaluator to incorporate emotional significance — not just raw intensity — into its routing decision.

§ 03

The MPCS Framework

MPCS is a unified architecture that integrates all four background concepts into a coherent cognitive processing loop. Its defining insight is borrowed from dual-process theory in cognitive psychology: System 1 is fast, automatic, and emotionally driven; System 2 is slow, deliberate, and analytical. MPCS provides both in a single, interoperable framework.

The five pillars of MPCS are:

Pillar 1
Malleable input
Structured perception that works even with missing modalities.
Pillar 2
Salience routing
Dynamic threshold that decides how much cognition to allocate.
Pillar 3
Dual processing
A reflex path and a deliberative path, both feeding a shared output.
Pillar 4
Unified memory
Fast membership layer + rich structured experience store.
Pillar 5
Adaptive learning
Outcomes update memory, improving future routing and decisions.

§ 05

Malleable Sensory Input Design

What it is and why it matters

The first stage of MPCS transforms raw environmental signals into a structured, normalized perception record. The word malleable is the key: the input format is designed to remain functional even when some modalities are missing or corrupted — just as a person can still perceive danger when visibility is low, using sound and body tension alone.

Each percept is represented as a compound record with three layers: raw sensory modalities, derived cognitive features, and contextual metadata.

Diagram 2 — Sensory Input Structure

The malleability property

The defining property of MPCS sensory input is that it degrades gracefully. If a fire alarm sounds in complete darkness, the vision modality is empty — but the audio modality carries extreme intensity and high arousal, which is sufficient to trigger a high-salience reflex response. The system does not fail; it adapts. This mirrors how biological organisms respond: a person with impaired vision still reacts appropriately to sudden loud sounds.

Multisensory encoding format

Each individual modality reading is stored as a triple:

( feature, value, emotional_tag ) Example (audio channel): ("amplitude", 0.94, "startle") Example (vision channel): ("object_class", "fire", "threat")

This encoding ensures that emotional significance is bound to the feature at the lowest level, not inferred only at a higher stage — enabling faster and more accurate salience computation.

§ 06

Salience & Threshold Mechanism

What is salience?

Salience is the system's measure of how much attention a stimulus deserves. Not all incoming stimuli are equal: a background hum deserves little attention; a sudden scream deserves immediate full focus. The salience score aggregates four independent signals into a single priority value.

The four salience components

ComponentWhat it measuresBiological analogue
IntensityRaw magnitude of the stimulus (volume, brightness, pressure)Loudness triggering the acoustic startle reflex
ArousalThe degree to which the stimulus activates or excites the systemAdrenaline response to perceived threat
|Valence|Absolute emotional significance (strongly positive or strongly negative)Fear or euphoria — both demand attention
NoveltyHow different this stimulus is from anything previously encounteredOrienting response to an unfamiliar stimulus

The salience formula

Salience = w₁·Intensity + w₂·Arousal + w₃·|Valence| + w₄·Novelty where w₁ + w₂ + w₃ + w₄ = 1, and weights are learned from experience

Note the use of absolute value on valence: both strong fear and strong joy are highly salient. A wedding announcement is not an emergency, but it is unusual and emotionally loaded — the system should pay attention, though not reflexively. This is captured correctly only by using |Valence| rather than Valence.

Threshold and routing

Once salience is computed, it is compared against a learned threshold θ. This threshold is not fixed — it adapts based on experience and context. In a high-risk environment (a construction site), θ lowers, making the system more trigger-happy for reflex responses. In a safe context (a library), θ rises, sending more stimuli through deliberative reasoning.

Diagram 3 — Salience Decision Flow
Key design insight

The threshold θ is a learned, context-sensitive parameter — not a static constant. In dangerous environments, it decreases (more stimuli trigger reflex responses). In safe, familiar environments, it increases (more stimuli flow to deliberative reasoning). This adaptability is what makes MPCS genuinely responsive to environmental context.

§ 07

Reflex Pathway

Purpose

The reflex pathway handles the high-salience, time-critical subset of stimuli. Its design philosophy is speed over precision: when something demands immediate action — a threat, an extreme pain signal, a sudden collision — the cost of deliberation is too high. The system must act before it finishes thinking.

Biologically, this corresponds to subcortical reflex arcs: the spinal cord can trigger a withdrawal response to burning heat before the signal even reaches the cortex for conscious processing. MPCS implements the computational equivalent.

Mechanism

Reflex responses are predefined stimulus-action mappings. Rather than querying memory or reasoning through context, the system looks up a pre-established table of high-urgency stimulus patterns and their corresponding immediate responses.

Stimulus typeReflex responseProperties
Sudden loud noise (intensity > 0.9)Startle response, alert posture~2ms latency, no reasoning
Heat/pain signal (internal state critical)Immediate withdrawal from sourceSub-cortical equivalent
High-speed approaching object (visual threat)Evasive movement initiationBypasses semantic analysis
System anomaly (critical error state)Safe shutdown or failsafe modePriority interrupt

Properties of the reflex path

Speed
Minimal latency
Fixed lookup table — O(1) response time. No iterative reasoning.
Computation
Low overhead
No semantic matching, no memory retrieval, no probabilistic inference.
Tradeoff
Reduced precision
Responses may be sub-optimal for edge cases — this is acceptable given that safety takes precedence.
Design note

The reflex pathway still feeds into the Experience Storage system after the response is executed. This means that even fast, automatic responses become part of the learning signal. Over time, the system can detect whether its reflexes are well-calibrated — and flag cases where more deliberation would have led to better outcomes, potentially adjusting the threshold θ.

§ 08

Deliberative Pathway

When deliberation is triggered

The deliberative path is activated for stimuli that are ambiguous, moderately novel, or below the reflex threshold. These are the situations where fast heuristics are unreliable and considered reasoning produces a better outcome. The deliberative pathway is the MPCS equivalent of System 2 cognition — slow, effortful, and context-sensitive.

Diagram 4 — Deliberative Pathway: Processing Steps

Step-by-step breakdown

Step 1 — Semantic matching

The incoming stimulus is compared against known concept representations. This is not just pattern matching — it draws on a semantic embedding that encodes meaning, not just surface features. A hissing sound is compared against concepts like "kettle", "snake", "deflating tire" and ranked by embedding similarity.

Step 2 — Emotional alignment

The stimulus's valence and arousal profile is compared against emotional patterns stored in memory. Prior experiences with high-arousal, negative-valence outcomes make the system more cautious about similarly profiled stimuli.

Step 3 — Memory retrieval

The structured memory layer is queried for past experiences that share similar sensory fingerprints, semantic labels, and emotional profiles.

Step 4 — Memory borrowing

Rather than relying on a single best-matching memory, MPCS uses the top-K most similar experiences and combines their outcomes using a weighted vote:

Action estimate = Σᵢ sim(S, Eᵢ) · outcome(Eᵢ) for i in top-K similar experiences sim(·,·) = cosine similarity in embedding space

Step 5 — Contextual reasoning

The belief, context, and emotional state (encoded as Z* numbers) are integrated with the memory estimates to produce a final probabilistic assessment of the best action.

Step 6 — Decision

The action with the highest expected utility, given the reasoning above, is selected and executed.

§ 09

Memory System Design

MPCS employs a two-layer memory architecture that mirrors the complementary memory systems found in biological organisms: a fast, coarse recognition layer and a rich, detailed associative store.

Diagram 5 — Memory Architecture (Two-Layer)

Layer 1 — Fast membership

The fast membership layer answers a binary question with near-zero latency: has this stimulus been encountered before? It uses a Bloom-filter-style hash structure to map incoming stimuli to a compact bit array. If the answer is "no", the system flags this as a novel stimulus and raises its novelty component in the salience formula. If the answer is "yes", the system proceeds to the structured store for richer retrieval.

This layer operates in constant time and requires minimal memory — it is designed to be the first port of call for every single incoming percept, so it must be as fast as possible.

Layer 2 — Structured experience store

Each entry in the structured store is a rich experience record:

Experience E = { sensory_pattern: compressed embedding / fingerprint, semantic_label: what this stimulus was identified as, valence: emotional polarity at time of encounter, arousal: activation level at time of encounter, context: environmental situation during encoding, action: what the system did in response, outcome: the result of that action (reward/punishment) }

Raw sensory data is never stored directly — only compressed fingerprints and embeddings. This dramatically reduces memory consumption and, importantly, enables fast semantic similarity search: to find the top-K experiences most similar to a new stimulus, the system compares embedding vectors rather than pixel arrays.

§ 10

Experience Encoding & Learning

The learning loop is what distinguishes MPCS from a static perception system. After every action — whether reflex or deliberative — the system encodes the experience and uses the outcome to update its internal state.

The encoding loop

Loop: Input → Process → Action → Outcome → Memory Update → (repeat)

Learning signals

Signal typeSystem responseLong-term effect
Reward (positive outcome)Reinforce the action policyIncreases probability of same action in similar context
Punishment (negative outcome)Penalize the action policyDecreases probability; considers alternatives
Novelty (unseen stimulus)Triggers exploration modeAdds new experience to the structured store
Repetition (seen many times)Strengthens existing memory traceFaster retrieval; reduces deliberation time for familiar patterns

What updates

The learning loop modifies three things simultaneously:

  • The structured memory store — new experiences are added; existing ones are updated with outcome data.
  • The salience weights (w₁–w₄) — weights are adjusted based on which components were most predictive of good/bad outcomes.
  • The threshold θ — recalibrated based on the frequency and severity of misrouting events (e.g., stimuli that should have been reflexive but went through deliberation, or vice versa).
Adaptive intelligence

This triple-update mechanism means that MPCS becomes more efficient over time: familiar situations are handled faster (lower deliberation cost), novel situations trigger appropriate exploration, and dangerous situations become more reliably routed to the reflex path. The system learns not just what to do, but how urgently to do it.

§ 11

End-to-End Processing Pipeline

The complete MPCS processing cycle, from raw sensation to stored experience, proceeds in seven stages.

Diagram 6 — End-to-End Processing Pipeline
  1. Capture — Raw sensory signals are sampled across all available modalities. Missing modalities leave their slots as null without breaking the pipeline.
  2. Structure — Raw signals are normalized, tagged with emotional annotations, and assembled into the SensoryInput record format.
  3. Score — The salience formula is evaluated. The fast membership layer simultaneously answers the novelty query.
  4. Route — The salience score is compared against the learned threshold θ to determine which processing pathway is engaged.
  5. Process — Either the reflex table is consulted for a fast action, or the deliberative pathway executes its multi-step reasoning chain.
  6. Act — The selected action is executed in the environment.
  7. Learn — The outcome is observed, the experience is encoded in the structured store, and the salience weights and threshold are updated.
§ 12

Key Contributions

MPCS makes five primary technical contributions to the design of cognitive AI systems:

  • Malleable multisensory input representation. A unified, fault-tolerant input format that gracefully handles missing or degraded modalities. Each modality is encoded with its own emotional tag, enabling bottom-up emotional integration rather than post-hoc inference.
  • Salience-based dual-path routing. A principled, learnable mechanism that dynamically allocates cognitive resources. The threshold θ is not a hyperparameter — it is a learned, context-sensitive quantity that adapts to the system's operational environment. This is the first explicit formalization of such a routing mechanism in the context of unified AI cognition.
  • Emotion, novelty, and context as first-class signals. Conventional AI systems treat emotional valence and novelty as secondary outputs of reasoning. MPCS treats them as primary inputs to the routing decision. This reflects what neuroscience tells us about the amygdala's role in prioritizing emotionally significant stimuli for fast processing.
  • Two-layer memory architecture. The combination of a fast Bloom-filter membership layer and a rich structured experience store allows the system to answer "have I seen this?" in O(1) time and "what should I do about it?" through semantic retrieval. These two operations are decoupled, making the system scalable.
  • Experience-driven adaptive learning. Every action — reflex or deliberative — produces a learning signal that updates three aspects of the system simultaneously: the memory store, the salience weights, and the routing threshold. This triple-update mechanism creates a feedback loop that improves both accuracy and efficiency over time.
§ 13

Discussion & Future Work

Limitations of the current framework

The current MPCS framework is a design-level specification. Several implementation challenges remain unresolved:

The salience weights w₁–w₄ require initialization before learning can begin, and a poor initialization may lead to suboptimal routing early in the system's lifetime. The threshold θ update rule has not been formally specified — a reinforcement learning formulation would be a natural fit, but the reward signal for routing decisions is non-trivially defined (how do you know a routing decision was wrong before you observe a long-term outcome?).

The deliberative pathway's memory borrowing step assumes a meaningful embedding space exists. For modalities like olfaction or proprioception, standard embedding methods are underdeveloped.

Future extensions

Extension 1
Multi-modal deep learning
Replace handcrafted feature extraction with learned multimodal encoders (e.g., vision-language models extended to touch and proprioception).
Extension 2
RL-based policy optimization
Replace the memory borrowing heuristic with a full reinforcement learning policy, where the deliberative path learns to select actions via gradient updates.
Extension 3
Memory decay mechanisms
Implement Ebbinghaus-style forgetting curves so that old, low-utility experiences fade over time, preventing memory saturation.
Extension 4
Attention-based filtering
Use transformer-style attention over the experience store to weight retrieved memories by contextual relevance rather than raw embedding similarity.
Extension 5
Extended emotional dimensions
Move beyond valence/arousal to a richer emotional representation (Plutchik's wheel or PAD model) to capture nuanced affective states like disgust, surprise, or anticipation.

Relationship to existing architectures

MPCS shares conceptual lineage with Global Workspace Theory (Baars, 1988), which posits a "global workspace" that broadcasts information to specialized processors. The salience module plays an analogous role. It also relates to the ACT-R cognitive architecture (Anderson, 2004), particularly the distinction between procedural memory (analogous to the reflex table) and declarative memory (analogous to the structured experience store). However, MPCS is distinctive in its explicit formalization of the routing decision as a learnable, context-sensitive threshold function — a feature absent from prior cognitive architectures.

§ 14

Conclusion

The Malleable Perceptual-Cognitive System presents a unified answer to a fundamental question in AI design: how should an intelligent system decide how much to think?

The answer MPCS proposes is principled and biologically grounded. By computing a salience score that combines raw intensity, emotional arousal, affective significance, and novelty, and comparing it against a learned, context-sensitive threshold, the system routes each percept to the processing pathway most appropriate to its urgency. High-stakes, time-critical stimuli bypass deliberation and trigger fast reflex responses. Ambiguous, moderately novel stimuli are routed to a multi-step reasoning chain that draws on memory, semantic matching, and contextual inference. All outcomes — from both paths — are recorded and used to continuously calibrate the system's sensitivity, efficiency, and decision quality.

The central design principle of MPCS is worth restating plainly:

Not all stimuli require equal thinking. Intelligent systems must decide how much cognition to allocate — and that decision itself must be learned, adaptive, and emotionally informed.

MPCS operationalizes this principle into a concrete, implementable framework. It does not merely describe dual-process cognition — it provides the architectural machinery to realize it in an AI system that perceives, evaluates, acts, and learns.
